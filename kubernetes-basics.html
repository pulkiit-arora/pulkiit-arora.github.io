üîß Core Concepts

üì¶ Pods - The Smallest Deployable Units

Pods are the smallest deployable units in Kubernetes. They can contain one or more containers and share the same network namespace.

apiVersion: v1 kind: Pod metadata: name: my-app-pod spec: containers: - name: my-app image: nginx:latest ports: - containerPort: 80

üöÄ Deployments - Managing Pod Lifecycle

Deployments provide declarative updates for Pods and ReplicaSets, enabling rolling updates and rollbacks.

apiVersion: apps/v1 kind: Deployment metadata: name: my-app-deployment spec: replicas: 3 selector: matchLabels: app: my-app template: metadata: labels: app: my-app spec: containers: - name: my-app image: nginx:latest

üîó Services - Network Abstraction

Services define a logical set of Pods and a policy by which to access them, providing stable IP addresses and DNS names.

apiVersion: v1 kind: Service metadata: name: my-app-service spec: selector: app: my-app ports: - protocol: TCP port: 80 targetPort: 80 type: ClusterIP
